## Technical Design

### Overview

This change has two scopes:

1. **CAF Naming Alignment**: Flip resource naming from `{project}-{env}-{type}` suffix pattern to `{type}-{project}-{env}` prefix pattern per Azure Cloud Adoption Framework.
2. **Backend Randomization**: Make `bootstrap-backend.sh` generate `backend.tf` with a random-suffixed storage account name (`tfstate$RANDOM`) per Azure documentation, and remove dead `.env.local` generation.

### Scope 1: CAF Resource Naming

#### Current vs CAF Naming

| Resource           | Current Pattern                          | CAF Pattern                              |
|--------------------|------------------------------------------|------------------------------------------|
| Resource Group     | `{project}-{env}-rg`                     | `rg-{project}-{env}`                     |
| ML Workspace       | `{project}-{env}-mlw`                    | `mlw-{project}-{env}`                    |
| Application Insights | `{project}-{env}-ai`                   | `appi-{project}-{env}`                   |
| Key Vault          | `{project}-{env}-kv-{suffix}`            | `kv-{project}-{env}-{suffix}`            |
| Storage Account    | `{project}{env}{suffix}sa`               | `st{project}{env}{suffix}`               |

#### Changes to `infrastructure/terraform/main.tf`

Update the `locals` block:

```hcl
locals {
  resource_group_name = "rg-${var.project_name}-${var.environment}"
  ml_workspace_name   = "mlw-${var.project_name}-${var.environment}"

  # Storage: CAF prefix "st", no hyphens, random suffix, max 24 chars
  storage_account_name = substr(
    replace("st${var.project_name}${var.environment}${random_string.suffix.result}", "-", ""),
    0,
    24
  )

  common_tags = merge(
    var.tags,
    {
      Environment = var.environment
      ManagedBy   = "terraform"
      Project     = var.project_name
    }
  )
}
```

Update inline resource names:
- Application Insights: `"appi-${var.project_name}-${var.environment}"`
- Key Vault: `substr("kv-${var.project_name}-${var.environment}-${random_string.suffix.result}", 0, 24)`

#### Changes to `README.md`

Update the "Resource Naming Convention" section to reflect the new CAF prefix pattern and examples.

### Scope 2: Backend Storage Randomization

#### Design Decision: `backend.tf` as Generated Artifact

The bootstrap script becomes the owner of `backend.tf`. The flow:

```
bootstrap-backend.sh
‚îú‚îÄ‚îÄ Does backend.tf exist?
‚îÇ   ‚îú‚îÄ‚îÄ YES ‚Üí Parse storage_account_name from it
‚îÇ   ‚îÇ         ‚îú‚îÄ‚îÄ Account exists in Azure? ‚Üí Use it (no-op)
‚îÇ   ‚îÇ         ‚îî‚îÄ‚îÄ Account missing? ‚Üí Create it with that name
‚îÇ   ‚îî‚îÄ‚îÄ NO  ‚Üí Generate name: tfstate$RANDOM
‚îÇ             ‚îú‚îÄ‚îÄ Create Azure resources (RG, account, container)
‚îÇ             ‚îî‚îÄ‚îÄ Generate backend.tf with the new name
‚îî‚îÄ‚îÄ Configure RBAC role assignment
```

#### Parsing `backend.tf`

Use `grep` to extract the storage account name:

```bash
STORAGE_ACCOUNT=$(grep 'storage_account_name' "$BACKEND_TF" | sed 's/.*"\(.*\)".*/\1/')
```

#### Generating `backend.tf`

When no `backend.tf` exists, generate it:

```bash
cat > "$BACKEND_TF" << EOF
terraform {
  backend "azurerm" {
    resource_group_name  = "${RESOURCE_GROUP}"
    storage_account_name = "${STORAGE_ACCOUNT}"
    container_name       = "${CONTAINER}"
    use_azuread_auth     = true
    # key is provided at init time:
    # terraform init -backend-config="key=<environment>.terraform.tfstate"
  }
}
EOF
```

#### Resource Group Naming

The backend resource group also aligns with CAF: `rg-tfstate-iemlops-<location>` (prefix pattern, scoped to project and region).

#### Storage Account Naming

Per Azure docs for Terraform state storage:
- Pattern: `tfstate$(date +%s)` (Unix epoch timestamp)
- Example: `tfstate1742169600`
- Globally unique, lowercase, no hyphens, max 24 chars

#### Removing `.env.local` Generation

- Remove the entire `.env.local` creation block (steps 3.8) from `bootstrap-backend.sh`
- Remove the `sed` commands that update `.env.local`
- Keep `.env.local` in `.gitignore` (defensive ‚Äî in case future use)

#### Changes to `scripts/bootstrap-backend.sh`

1. Remove hardcoded `STORAGE_ACCOUNT="tfstateiemlops"`
2. Add `BACKEND_TF="infrastructure/terraform/backend.tf"` path constant
3. Add logic to parse existing `backend.tf` or generate a new name
4. Replace resource group name with `rg-tfstate-iemlops-<location>` (CAF-aligned, project and region scoped)
5. Add `backend.tf` generation after successful resource creation
6. Remove `.env.local` generation block entirely
7. Update completion summary to reflect new behavior

#### Changes to `README.md`

- Update Backend Configuration section: document that `backend.tf` is generated by bootstrap
- Remove references to `.env.local` containing backend configuration
- Update backend resource names (`rg-tfstate-iemlops-<location>` instead of `terraform-state-rg`)
- Update storage account naming explanation (random suffix)

### Risk Assessment

- **Breaking change for existing deployments**: Terraform will see renamed resources as destroy+create. This is acceptable for greenfield or dev environments. For existing prod, a `terraform state mv` migration would be needed.
- **Backend rename**: Changing the state RG from `terraform-state-rg` to `rg-tfstate-iemlops-<location>` means existing state storage would need migration. The bootstrap script handles this by checking if `backend.tf` already exists ‚Äî existing deployments are unaffected.
