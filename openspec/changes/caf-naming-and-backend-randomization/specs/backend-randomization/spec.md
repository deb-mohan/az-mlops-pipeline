## MODIFIED Requirements

### Requirement: Bootstrap generates backend.tf with random storage account name
The system SHALL generate `infrastructure/terraform/backend.tf` with a storage account name using the pattern `tfstate$RANDOM`.

#### Scenario: First-time bootstrap (no backend.tf)
- **WHEN** developer runs bootstrap and `backend.tf` does not exist
- **THEN** system generates a storage account name `tfstate$RANDOM`, creates Azure resources, and writes `backend.tf` with that name

#### Scenario: Backend.tf already exists with valid account
- **WHEN** developer runs bootstrap and `backend.tf` exists with a storage account name that exists in Azure
- **THEN** system uses the existing account name and skips resource creation (idempotent)

#### Scenario: Backend.tf exists but account missing in Azure
- **WHEN** developer runs bootstrap and `backend.tf` exists but the storage account does not exist in Azure
- **THEN** system creates the storage account using the name from `backend.tf`

### Requirement: Backend resource group uses CAF naming
The system SHALL name the backend resource group using CAF prefix convention.

#### Scenario: New bootstrap creates CAF-named resource group
- **WHEN** bootstrap creates a new resource group for state storage
- **THEN** the resource group name is `rg-tfstate-iemlops-<location>` (e.g., `rg-tfstate-iemlops-eastus`)

### Requirement: Bootstrap does not generate .env.local
The system SHALL NOT create or update `.env.local` during bootstrap.

#### Scenario: Bootstrap completes without .env.local
- **WHEN** developer runs bootstrap to completion
- **THEN** no `.env.local` file is created or modified

### Requirement: .env.local remains in .gitignore
The system SHALL keep `.env.local` in `.gitignore` as a defensive measure.

#### Scenario: .gitignore preserves .env.local entry
- **WHEN** changes are applied
- **THEN** `.gitignore` still contains `.env.local`

### Requirement: README reflects updated backend configuration
The system SHALL update README.md to document the generated backend.tf and remove .env.local references.

#### Scenario: Backend documentation updated
- **WHEN** a developer reads the Backend Configuration section
- **THEN** it documents that `backend.tf` is generated by bootstrap with a random storage account name

#### Scenario: .env.local references removed from README
- **WHEN** a developer reads the README
- **THEN** there are no references to `.env.local` containing backend configuration metadata
